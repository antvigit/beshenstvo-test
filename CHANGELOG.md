# Changelog

Все заметные изменения в проекте задокументированы здесь.

## [v2.5] — 2026-06-25
### Изменено
- **Отказ от Gunicorn:** бот снова работает на встроенном Flask-сервере с `app.run()`.
- **Режим работы:** переключились на **polling** с параллельным health check для Render.
- Удалены зависимости от Gunicorn из `requirements.txt`.
- Обновлена команда запуска на Render: `python bot/bot.py`.

### Исправлено
- Бот снова стабильно отвечает на команды (проблемы с вебхуком устранены отказом от него).

## [v2.4] — 2026-06-25
### Изменено
- **Production‑сервер для Telegram‑бота:** встроенный Flask‑сервер заменён на **Gunicorn**. Бот теперь работает стабильнее и не выдаёт предупреждение о development‑сервере.
- Обновлён `requirements.txt` (добавлен `gunicorn`).
- Обновлена команда запуска на Render (`gunicorn bot:app`).

## [v2.3] — 2026-06-25
### Добавлено
- **Контейнеризация тестов:** добавлен `Dockerfile` для запуска тестов в контейнере. Теперь тесты можно запускать в любой среде, где есть Docker, без установки Java, Maven и Chrome.
- **Полное логирование:** все `System.out.println` заменены на Log4j (`log.info`, `log.warn`). Добавлен файл `log4j2.xml`.
- **Параллельный запуск тестов внутри одного браузера:** включён параллелизм JUnit 5 через `junit-platform.properties`. Тесты теперь выполняются быстрее.

### Исправлено
- Отправка сообщений в Telegram теперь использует `--data-urlencode` для корректного переноса строк в caption.

## [v2.2] — 2026-06-25
### Добавлено
- Health check эндпоинт `/health` для Render (бот не засыпает на бесплатном тарифе).
- Логирование через Log4j вместо `System.out.println`.
- Конфигурация `log4j2.xml` для красивого вывода логов в консоль.

### Исправлено
- Заменены все `System.out.println` в тестах на `log.info` и `log.warn`.

## [v2.1] — 2026-06-25
### Добавлено
- Docker Compose для поднятия Selenium Grid (Chrome + Firefox).
- Поддержка параллельного запуска тестов в двух браузерах через матрицу в GitHub Actions.
- Отдельные Allure-отчёты и скриншоты для каждого браузера.
- Бот переведён на вебхук (Flask) для стабильной работы (устранена ошибка 409).
- Удалены лишние сообщения от бота («скриншот уже в пути...», персональные завершения).
- Отправка отчётов в чат пользователя, запустившего тесты.

### Исправлено
- Ошибка `tab crashed` в Chrome при работе через Grid (добавлены headless-аргументы и увеличен `shm_size`).
- Ошибка `docker-compose` заменена на `docker compose`.
- Предупреждения Node.js 20 в GitHub Actions (добавлена переменная окружения).
