# Beshenstvo Test — автотесты для калькулятора прививок

Автотесты для сайта [beshenstvo.pro](https://beshenstvo.pro/) — проверка корректности отображения дат вакцинации.

Проект включает:
- Автотесты на Java + Selenium + JUnit 5.
- Allure-отчёты.
- Telegram-бота для удалённого запуска тестов и получения отчётов.

## 🧱 Архитектура проекта: ООП, POM, SOLID

Проект построен на фундаментальных принципах:

**1. Объектно-ориентированное программирование (ООП):**
- **Наследование:** `VaccinationPage` расширяет `BasePage`.
- **Инкапсуляция:** Элементы страницы (`WebElement`) — `private`, доступ через публичные методы.
- **Полиморфизм:** Используются интерфейсы `IPage` и `WebDriver` для гибкой замены реализаций.

**2. Page Object Model (POM) + Page Factory:**
- Каждая страница — отдельный класс (`VaccinationPage`) с локаторами и методами.
- Для инициализации элементов используется `@FindBy` (Page Factory).

**3. Принципы SOLID:**
- **S** — каждый класс отвечает за одну задачу.
- **O** — добавление новой страницы без изменения старого кода.
- **L** — замена драйвера (Chrome → Firefox) без правок в тестах.
- **I** — интерфейс `IPage` содержит только 2 метода.
- **D** — зависимость от абстракции `WebDriver`.

## 🧪 Тесты

Проект содержит один основной тест:

|ест	Что проверяет	Статус
shouldValidateVaccinationDates	Наличие дат на странице, формат ДД.ММ.ГГГГ, День, хронологический порядок и правильные интервалы (3, 4, 7, 16, 60 дней)	✅ Проходит
testValidNameCyrillic	Ввод имени на кириллице	❌ Падает
testEmptyName	Ошибка при пустом поле	❌ Падает

## 🤖 Telegram-бот

Бот позволяет запускать тесты удалённо и получать результат в Telegram.

**Команды:**

- `/start` — приветствие.
- `/run` — запустить тесты.

**Что приходит:**

1. Статус запуска (🔄)
2. Ожидание завершения (⏳)
3. Скриншот Allure-отчёта (📊)
4. Ссылка на полный отчёт в GitHub Actions (🔗)
5. Финальное сообщение с результатом (✅)
